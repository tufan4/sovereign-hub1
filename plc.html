⚡ Bölüm 1

Temel Komutlar & Lojik Mantık

S7-200 PLC'nin çekirdek işletim kuralları — kontaklar, çıkışlar, lojik yığın ve tarama döngüsü ile entegre temel yapı taşları.

🔘 Normalde Açık (NO) & Normalde Kapalı (NC) Kontaklar

Tüm lojik devrenin başlangıç noktasıdır. Bir adres biti 1 olduğunda NO kontak kapanır; NC kontak açılır.

Kontak Adres = 0 Adres = 1 Ladder Simgesi
NO (Normalde Açık) Açık → enerji geçmez Kapanır → enerji geçer ─┤ ├─
NC (Normalde Kapalı) Kapalı → enerji geçer Açılır → enerji kesilir ─┤/├─
💡
Operant Esnekliği
Her iki kontak da I, Q, M, SM, S, T, C, V bit adreslerini operant olarak kabul eder. Bu sayede fiziksel bir sensörle zamanlayıcı biti aynı lojik katmanda kullanılabilir.
🔗
Bağlantı → Lojik
Seri bağlantı = AND (Ve) — iki şart birden sağlanmalı.
Paralel bağlantı = OR (Veya) — şartlardan biri yeterli.
🚫
NOT Kontağı Kısıtlaması
NOT kontağı hiçbir zaman doğrudan sol güç hattına bağlanamaz ve paralel kollar arasına yerleştirilemez. Benzer kural P ve N kenar tetikleme kontakları için de geçerlidir.
🔲 AND & OR Lojik Tablosu

🔒 Set (S) / Reset (R) — Kalıcı Lojik

Klasik kontaktörlü devrelerdeki "mühürleme (latching)" mantığının PLC karşılığıdır. Şart ortadan kalktığında çıkış durumunu korur.

Komut Davranış Kapasite Özel Not
S bit, N N adet biti kalıcı '1' yapar 1–255 bit Şart kaybolsa da '1' kalır
R bit, N N adet biti kalıcı '0' yapar 1–255 bit T ve C adreslerini de sıfırlar
SI bit, N Tarama döngüsünü beklemeden anında set 1–64 bit Sadece Q alanı
RI bit, N Tarama döngüsünü beklemeden anında reset 1–64 bit Sadece Q alanı
⚡
SI/RI — Acil Durum Komutları
Normal PLC döngüsü: program tara → lojik çöz → çıkışları güncelle. SI/RI bu döngüyü yıkarak çıkışı o satır okunduğu anda fiziksel donanıma yansıtır. Pres makineleri, yüksek hızlı kesiciler gibi senaryolarda kullanılır. Standart S/R'ye kıyasla 255 değil 64 bit kapasitesi ve yalnızca Q alanı kısıtı vardır.
⬆ Kenar Tetikleme — P (Pozitif) & N (Negatif)

Sinyalin ne kadar süre var olduğu değil, ne zaman değiştiği önemlidir. PLC programı saniyede binlerce kez taranır — kenar tetikleme olmadan sayıcılar anında hatalı sayar.

P — Pozitif Kenar (0→1 Geçişi)

Sensör ilk kez algılama yaptığında tek bir tarama döngüsü boyunca '1' üretir. Ürün sensörün önünde saatlerce beklese bile sayıcıya giden sinyal kesilir → 1 adet sayılır.

N — Negatif Kenar (1→0 Geçişi)

Sinyal kesildiği anı yakalar. Ürün sensörün önünden tamamen çıktığında tek darbe üretir. Presin inmesi veya kapağın kapanması gibi ardışık işlemlerde kullanılır.

🚫
CPU Başına Donanımsal Limit
CPU 210: 32 adet  |  CPU 212: 128 adet  |  CPU 214–226: 256 adet kenar tetikleme elemanı kullanılabilir. P ve N kontakları güç hattına doğrudan bağlanamaz ve paralel kollar arasına yerleştirilemez.

🏭 Endüstriyel Senaryo

Motor Start / Stop Devresi

Bir konveyör motoru: I0.0 = Start butonu, I0.1 = Stop butonu (NC), I0.2 = Aşırı akım koruması (NC). Çıkış: Q0.0 = Motor kontaktörü.

🪜 Ladder (LAD)
📋 STL (Komut Listesi)
STL 
// Network 1: Motor Başlatma
LD    I0.0   // Start butonu (NO)
O     Q0.0   // Mühürleme (self-hold)
AN    I0.1   // Stop butonu (NC → AN)
AN    I0.2   // Aşırı akım koruma (NC)
=     Q0.0   // Motor çıkışı

📋 Bölüm 1 — Özet Kartı

NO: Adres=1 → kapanır, enerji geçer
NC: Adres=1 → açılır, enerji kesilir
Seri = AND, Paralel = OR
NOT güç hattına doğrudan bağlanamaz
P/N kenar: tek tarama döngüsü enerji
S/R: 1–255 bit kalıcı; SI/RI: 1–64 bit, sadece Q
LD/LDN: yığına yükleme, dal başlatma
ALD/OLD: yığın derinliği 1 azaltır

📌
Dizilim Kuralı (Endianness)
Word veya Double Word hafızaya yerleşirken: byte/word numarası küçük olan → yüksek değerli (MSB), büyük olan → düşük değerli (LSB) konumlanır. Bu kural M, V ve tüm geniş alanlarda geçerlidir.

📦 MOV Komutları — Veri Taşıma

Veri boyutu, kullanılacak MOV varyantını belirler. Boyut uyumsuzluğu komutu devre dışı bırakır.

Komut Boyut Kaynak (IN) Hedef (OUT) Tipik Kullanım
MOV_B 8 bit VB, MB, IB… VB, MB, QB… Küçük tamsayı taşıma
MOV_W 16 bit VW, MW, AIW… VW, MW, AQW… Analog değer, timer PT taşıma
MOV_D 32 bit VD, MD, HC… VD, MD… Hızlı sayıcı değeri taşıma
MOV_R 32 bit reel VD (real)… VD (real)… Kayan noktalı hesaplama
⛔
EN Bacağı Doğrudan Güç Hattına Bağlanamaz!
MOV kutusunun EN girişi mutlaka bir kontak veya SM0.0 (daima 1) ile tetiklenmelidir. Aksi takdirde derleme hatası üretilir.

Sonraki Bağlantı: Zaman Röleleri

Zaman rölelerinin PT değerinin neden 32767 ile sınırlı olduğunu şimdi anlayabilirsiniz: Word alanında saklanır! Bölüm 3: Zaman Röleleri'ne geçin.

⏱ Bölüm 3

Zaman Röleleri (Timers)

TON, TOF ve TONR — geciktirme, kalıcı sayım ve zaman hesaplama formülleri.

🔑
Neden Maksimum 32767?
PT (Preset Time) değeri PLC hafızasında işaretli Word (16-bit) alanında saklanır. İşaretli Word'ün alabildiği maksimum pozitif değer = +32767. Bu donanımsal bir kısıtlamadır, rastgele seçilmiş bir sayı değildir.
🧮 PT Hesaplama Formülü
PT = İstenen Süre (ms) ÷ Zaman Tabanı
Sonuç 32767'yi geçemez. Uzun süreler için büyük taban seçin.
Zaman Tabanı Timer Numaraları Maks. Fiziksel Süre Örnek: 25 sn
1 ms T0, T32 (TON/TOF) | T0–T31 (TONR) 32,767 ms ≈ 32,7 sn 25000 / 1 = 25000 ✓
10 ms T1–T4, T33–T36 (TON/TOF) 327,67 sn ≈ 5,4 dk 25000 / 10 = 2500 ✓
100 ms T37–T63 (TON/TOF) | T101–T255 3276,7 sn ≈ 54,6 dk 25000 / 100 = 250 ✓
⚖ TON / TOF / TONR — Fark Nedir?
TON
Çekmede Gecikmeli

IN'e enerji gelince sayar. PT'ye ulaşınca çıkış verir.

⚡ Enerji kesilirse → anında sıfır
📌 Kullanım: "5 sn sonra motor çalışsın"
TOF
Düşmede Gecikmeli

IN'e enerji gelince çıkışı anında aktif eder, saymaz. Enerji kesilince sayar.

🔥 Enerji kesilince → sayım başlar
📌 Kullanım: "Motor durduktan 30 sn soğutma fanı çalışsın"
TONR
Kalıcı / Toplamalı

Enerji kesilse dahi sayma değerini hafızada korur. Gelince kaldığı yerden devam eder.

🧠 Enerji kesilirse → değer korunur
📌 Kullanım: "Motorun toplam çalışma saati" + Reset gerekli!
📈 Timer Davranış Zaman Çizelgesi
📋 Timer Komut Örnekleri
TON 
// 2.5 saniye gecikme
// T37 = 100ms taban
LD   I0.0
TON  T37, 25
// 25 × 100ms = 2500ms
LD   T37
=    Q0.0
TOF 
// Fan 5 sn daha çalışsın
// T38 = 100ms taban
LD   I0.1
TOF  T38, 50
// 50 × 100ms = 5000ms
LD   T38
=    Q0.1
TONR 
// Toplam çalışma saati
LD   I0.0   // Motor çalışıyor
TONR T4, 32767
// Resetleme:
LD   I0.5   // Bakım butonu
R    T4, 1
🏭 Endüstriyel Senaryo

Soğutma Sistemi — TOF Kullanımı

Bir CNC tezgahının soğutma pompası: tezgah STOP aldıktan sonra pompa 60 saniye daha çalışarak kesici ucu soğutmaya devam eder. I0.0 = Tezgah RUN sinyali, Q0.2 = Soğutma pompası.

STL — TOF Soğutma Devresi 
// T37 = 100ms tabanlı TOF, PT=600 → 60 saniye
LD    I0.0    // Tezgah RUN sinyali
TOF   T37, 600  // 600 × 100ms = 60 000ms = 60 sn
LD    T37     // Tezgah çalışırken ve kapandıktan 60 sn boyunca '1'
=     Q0.2    // Soğutma pompası çıkışı

📋 Bölüm 3 — Özet Kartı

PT maks: 32767 (Word sınırı)
Formül: PT = ms / Taban
TON: Enerji gelince say, kesilince sıfır
TOF: Enerji kesilince sayar, kapanış gecikmesi
TONR: Değeri korur, R komutuyla sıfırla
Tabanlar: 1ms / 10ms / 100ms
TONR zorunluluğu: Mutlaka R (Reset) bobini
100ms taban maks: ≈54,6 dakika

Sonraki Bağlantı: Sayıcılar

Sayıcılardaki PV sınırı da aynı nedenle 32767'dir. TONR'deki kalıcılık mantığı sayıcılarda da geçerli — Bölüm 4: Sayıcılar'a geçin.

🔢 Bölüm 4

Sayıcılar (Counters)

CTU, CTD ve CTUD — kenar tetikleme zorunluluğu, PV sınırı ve reset mekanizmaları.

📌
Genel Kural: CPU 224'te 256 Adet Sayıcı (C0–C255)
Tüm sayıcılarda PV maks = 32767 (Word alanı sınırı). Tüm sayıcılar enerjileri kesilse bile değerlerini korur — bu yüzden R (Reset) komutu zorunludur. Sayıcı girişlerine mutlaka P veya N kenar tetikleme konulmalıdır.
🔄 CTU / CTD / CTUD Karşılaştırma Tablosu
Özellik CTU (Yukarı) CTD (Aşağı) CTUD (İki Yönlü)
Başlangıç Değeri 0'dan başlar PV yüklenmeli (LD) 0'dan başlar
Sayma Yönü CU: 0 → PV CD: PV → 0 CU: +1 | CD: -1
Çıkış Koşulu Değer ≥ PV → '1' Değer = 0 → '1' Değer ≥ PV → '1'
Çıkış Kapanma R komutuyla R komutuyla CD ile PV'nin altına → '0' veya R
Reset Yöntemi R girişi R girişi R girişi
Ön Yükleme Gerekmez LD girişi zorunlu Gerekmez
Kenar Tetikleme CU önüne P/N CD önüne P/N CU ve CD önüne P/N
⬇ CTD — Neden LD (Load) Yüklemesi Zorunlu?

CTD geriye doğru saydığı için bir "başlangıç noktasına" ihtiyaç duyar. CD girişinden sayma başlamadan önce LD girişine sinyal gönderilerek PV değeri sayıcının içine yüklenir. Bu işlem genellikle SM0.1 (ilk tarama biti) ile otomatize edilir.

📋 CTD STL Örneği
STL 
// Network 1: İlk açılışta yükle
LD    SM0.1   // Sadece ilk taramada '1'
LD    SM0.1
CTD   C10, 5  // PV=5 yüklendi

// Network 2: Aşağı sayım
LD    I0.0
EU             // Pozitif kenar (P)
LD    SM0.1   // LD (Load) sinyali
CTD   C10, 5

// Network 3: Çıkış
LD    C10    // 0'a ulaşınca '1'
=     Q0.0
🪜 CTU Ladder Örneği
LAD — Konveyör Sayımı 
// 10 ürün sayılınca Q aktif

// Network 1: Sayım
[P] I0.0   // Ürün sensörü (P kenar)
   │
[CTU C0, PV=10]

// Network 2: Çıkış
[C0] ──────────── (Q0.0)
         // 10 ürün doldu

// Network 3: Reset
[P] I0.1   // Reset butonu
   │
[R C0, 1]

Otopark Kapasite Takibi — CTUD

I0.0 = Giriş sensörü (araç girdi), I0.1 = Çıkış sensörü (araç çıktı), PV=50 = Kapasite, Q0.5 = "DOLU" tabelası.

STL — CTUD Otopark 
// CU: araç girişi, CD: araç çıkışı
LD    I0.0
EU             // Giriş kenar tetikleme
LD    I0.1
EU             // Çıkış kenar tetikleme
LD    I0.2    // Reset (vardiya sonu)
CTUD  C0, 50  // PV=50 kapasite

LD    C0     // 50 araç dolunca '1'
=     Q0.5   // DOLU tabelası

📋 Bölüm 4 — Özet Kartı

PV maks: 32767 (Word sınırı)
P/N kenar: Sayıcı girişlerine zorunlu
CTU: 0'dan PV'ye, sıfır için R
CTD: LD yükleme zorunlu (SM0.1 ile)
CTUD: PV altına düşerse çıkış kapanır
Reset: Bit ve değer ikisi de '0' olur

20–30 KHz: Standart sayıcı yetmez → HC kullan
📚 Bölüm 5

STL Yığın (Stack) İşlemleri

Ladder diyagramlarının arka planında çalışan lojik yığın — derinlik yönetimi, dallandırma ve birleştirme.

🏗
Yığın (Stack) Nedir?
S7-200 işlemcisinde maksimum 9 kademe derinliğe sahip bir lojik yığın bulunur. Bit değerleri üst üste itilir, işlenir ve çekilir. Tüm LD, A, O, ALD, OLD, LPS işlemleri bu yığın üzerinde gerçekleşir.
📊 STL Yığın Komutları — Tam Referans
Komut İşlev Yığın Etkisi Dikkat
LD n Adresi yığına yükle (NO) Derinlik +1 ↑ (diğerleri kayar) Yeni dal başlatır
LDN n Adresi tersleyip yükle (NC) Derinlik +1 ↑ Yeni dal başlatır
ALD Üstteki 2 biti AND'le Derinlik -1 ↓ 2 bloğu seri birleştirir
OLD Üstteki 2 biti OR'la Derinlik -1 ↓ 2 bloğu paralel birleştirir
LPS Tepeyi kopyala, iter Derinlik +1 ↑ (alt bit kayar!) Çoklu çıkış ilk dalı
LRD 2. biti tepeye kopyala Derinlik değişmez Ara dallar için güvenli
LPP Tepe değeri at, 2. yukarı gel Derinlik -1 ↓ Son dalı kapatır
🌿 LPS / LRD / LPP — Çoklu Çıkış Dallandırması

Tek bir giriş şartından birden fazla çıkış kontrol edilmek istendiğinde kullanılır. Doğru sıralama kritiktir.

🪜 Ladder — 3 Çıkışlı Devre
📋 STL Karşılığı
STL 
// Ortak şart
LD    I0.0

// 3 çıkışa dağıt
LPS          // Yığın: [I0.0, I0.0] ↑
=     Q0.0   // 1. çıkış

LRD          // Yığın: değişmedi
=     Q0.1   // 2. çıkış

LPP          // Yığın: -1 ↓, dal kapandı
=     Q0.2   // 3. (son) çıkış
⚠
LPS Kullanımında Kapasite Uyarısı
S7-200 yığını maksimum 9 kademe derinliğindedir. Her LPS yığını bir kademe iter ve en alttaki bit dışarı itilerek kalıcı olarak kaybolur. Art arda çok fazla LPS kullanmaktan kaçının. Her LPS'nin mutlaka bir LPP ile kapatılması gerekir.
🔗 ALD / OLD — Blok Birleştirme

Karmaşık devrelerde bağımsız lojik bloklar ayrı ayrı oluşturulur, sonra ALD veya OLD ile birleştirilir.

📋 ALD — Seri Blok Birleştirme
STL 
// Blok 1: (I0.0 OR I0.1)
LD   I0.0
O    I0.1

// Blok 2: (I0.2 OR I0.3)
LD   I0.2
O    I0.3

// Blok 1 AND Blok 2
ALD           // Yığın -1
=    Q0.0
📋 OLD — Paralel Blok Birleştirme
STL 
// Blok 1: (I0.0 AND I0.1)
LD   I0.0
A    I0.1

// Blok 2: (I0.2 AND I0.3)
LD   I0.2
A    I0.3

// Blok 1 OR Blok 2
OLD           // Yığın -1
=    Q0.0

📋 Bölüm 5 — Özet Kartı

Yığın derinliği: Maks 9 kademe
LD/LDN: Yığın +1, yeni dal
ALD/OLD: Yığın -1, iki bloğu birleştir
LPS: Yığın +1, alt bit kaybolabilir
LRD: Yığın değişmez, güvenli ara dal
LPP: Yığın -1, son dalı kapatır
LPS'yi her zaman LPP ile kapat
LRD defalarca kullanılabilir
🚀 Bölüm 6

Gelişmiş Operasyonlar

Tarama döngüsünü aşan anında müdahaleler, veri taşıma kuralları, karşılaştırma komutları ve hata yönetimi.

⚖ Karşılaştırma Komutları — Tip Eşleşme Zorunluluğu

İki veri kıyaslanmadan önce boyutları ve tipleri birebir eşleşmelidir. Byte sadece Byte ile kıyaslanabilir.

Tip Boyut Karakter Maks Değer Negatif?
B (Byte) 8 bit İşaretsiz 255 Hayır
I (Word) 16 bit İşaretli 32.767 Evet
D (DWord) 32 bit İşaretli 2.147.483.647 Evet
R (Real) 32 bit IEEE 754 ±3.4×10³⁸ Evet
🌡
Pratik Örnek: Soğuk Oda Sıcaklık Kontrolü
Eksi derecelerdeki verileri işlerken Byte (B) tipi kullanılamaz — işaretsiz olduğu için negatif sayıyı temsil edemez. Mutlaka I (Word) veya R (Real) tipi seçilmelidir.
📦 FBD (Fonksiyon Blok Diyagramı) — MOV Örneği

FBD dilinde her işlem bir blok olarak gösterilir. Girişler soldan, çıkışlar sağdan bağlanır.

Yukarıdaki FBD: SM0.0 (daima 1) ile MOV_W etkinleştirilir → AIW0'dan analog değer VW100'e taşınır → CMP_W ile 500 değeriyle karşılaştırılır → eşit ise Q0.0 aktif.

🖊 ST (Yapılandırılmış Metin) — Karşılaştırma Örneği
ST (Structured Text) 
(* Analog sıcaklık sensörü kontrolü *)
VAR
  Sicaklik_Raw  : INT;      (* AIW0'dan okunan ham değer *)
  Alarm_Aktif   : BOOL;
  Fan_Cikis     : BOOL;
END_VAR

(* Analog değeri oku *)
Sicaklik_Raw := AIW0;   (* Word boyutu zorunlu *)

(* Karşılaştırma: Word (I) tipi — işaretli → negatif değer destekler *)
IF Sicaklik_Raw > 800 THEN
  Fan_Cikis := TRUE;
  Alarm_Aktif := FALSE;
ELSIF Sicaklik_Raw > 1200 THEN
  Fan_Cikis := TRUE;
  Alarm_Aktif := TRUE;   (* Kritik aşırı ısınma! *)
ELSE
  Fan_Cikis := FALSE;
  Alarm_Aktif := FALSE;
END_IF
🏭 Endüstriyel Senaryo — Acil Durum

Pres Makinesi — SI/RI Kullanımı

Durum: Pres kafası inmeye başladığında güvenlik sensörü tetiklenir. Normal tarama döngüsü beklenirse pres zararı önlemek için geç kalabilir. SI ile acil durdurma anında uygulanır.

STL — SI/RI Acil Durdurucu 
// NORMAL ÇALIŞMA
LD    I0.0    // Başlat butonu
S     Q0.0, 1 // Pres motor → Set

// ACİL DURDURMA — Tarama döngüsünü atlar!
LD    I0.7    // Güvenlik sensörü (acil)
SI    Q0.3, 1 // Alarm: ANINDA aktif (Q alanı, 1–64 bit)
RI    Q0.0, 1 // Motor: ANINDA durdur
// RI/SI → END komutunu beklemez, fiziksel çıkış anında güncellenir

📋 Bölüm 6 — Özet Kartı

SI/RI: Tarama beklemez, sadece Q, 1–64 bit
MOV EN: Güç hattına doğrudan bağlanamaz
Hata 0006: ENO=0 → geçersiz endirekt adres
B: İşaretsiz, negatif yok, maks 255
I/D/R: İşaretli, negatif desteklenir
S/R maks: 255 bit  |  SI/RI maks: 64 bit
FBD: Blok soldan-sağa, pin eşleşmeli
D/R karşılaştırma: Maks ±2.147.483.647
🔧 Bölüm 7

Özel Hafıza (SM) Bitleri

Harici sensöre gerek kalmadan PLC'nin kendi işletim sistemiyle kontrol — başlangıç bitleri, flaşörler ve hata kütükleri.

📊 SM Bit Referans Tablosu — Tam Liste
Bit İsim Davranış Kullanım Senaryosu
SM0.0 Daima 1 PLC RUN'da olduğu sürece sürekli '1' MOV/blok EN girişlerine sürekli tetikleme; güç hattına bağlanamayan komutların çalıştırılması
SM0.1 İlk Tarama Sadece ilk tarama döngüsünde '1', sonra '0' CTD LD yükleme, MOV ile başlangıç değerleri atama, sistem init
SM0.2 Kalıcı Veri Kaybı Kalıcı veri kaybı yaşandığında '1' Sistem sağlık kontrolü, alarm lojikleri
SM0.3 Güç Sonrası İlk Güç verildikten sonraki ilk taramada '1' Güç sonrası başlangıç prosedürleri
SM0.4 1 dk Flaşör 30 sn ON / 30 sn OFF Yavaş uyarı lambası, periyodik bakım hatırlatıcısı
SM0.5 1 sn Flaşör 0,5 sn ON / 0,5 sn OFF Standart uyarı lambası, göz dikkat çekici alarm
SM0.6 Tarama Sayacı 1 tarama ON / 1 tarama OFF Çok hızlı sinyaller, tarama bazlı olaylar
SM1.0 Sıfır Sonuç İşlem sonucu = 0 ise '1' Matematiksel sonuç kontrolü
SM1.1 Taşma/Geçersiz Kapasite aşımı veya geçersiz veri ise '1' Overflow koruması, hata yakalama
SM1.2 Negatif Sonuç İşlem sonucu negatif ise '1' İşaret kontrolü, alt sınır alarmı
SM1.3 Sıfıra Bölme Bölen = 0 ise '1' Kritik hata koruması, sistem güvenliği
⚡ SM0.0 vs SM0.1 — Sürekli mi, Tek Seferlik mi?
SM0.0 — Daima 1

PLC çalıştığı sürece hiç kesilmeden '1' üretir.

// MOV'u sürekli çalıştır
LD   SM0.0   // Daima 1
MOV_W AIW0, VW100
// Her taramada analog güncelle
SM0.1 — İlk Tarama

Yalnızca sistem ilk açıldığında '1' olur.

// CTD'ye başlangıç yükle
LD   SM0.1   // Tek seferlik
LD   SM0.1   // LD sinyali
CTD  C5, 100  // PV=100 yüklendi
🛡 SMB1 Hata Kütüğü — Matematiksel Güvenlik Ağı

PLC, matematiksel işlemlerde hata yaşandığında SMB1 kütüğünün ilgili bitini otomatik olarak '1' yapar. Programcı bu bitleri okuyarak sistemi koruyabilir.

⚠ Senaryo: Sıfıra Bölme Koruması

Üretim Hızı Hesaplama Devresi

Toplam ürün sayısı / çalışma süresi = hız. Ama çalışma süresi henüz 0 ise sistem kilitlenir.

STL — SM1.3 Korumalı Bölme 
// Bölme işlemi yap
LD    SM0.0
DIV_W VW0, VW2, VW10  // Ürün / Süre = Hız

// Sıfıra bölme yaşandı mı?
LD    SM1.3   // Sıfıra bölme flag
=     Q0.7    // HATA lambası aktif
R     Q0.0, 4 // Tüm motor çıkışları durdur
💡 SM0.4 / SM0.5 — Donanımsız Flaşör
📋 Klasik Yöntem (Karmaşık)
// Karmaşık timer ağı gerekir:
LD   I0.0
TON  T37, 5    // 0.5sn ON
LD   T37
TON  T38, 5    // 0.5sn OFF
LD   T38
R    T37, 1    // Sıfırla
LD   T37
=    Q0.0     // Lamba
✅ SM ile Basit Yöntem
// 1 satırla aynı sonuç:
LD   I0.0    // Alarm koşulu
A    SM0.5   // 0.5sn ON/OFF
=    Q0.0    // Lamba

// Yavaş uyarı (1 dk):
LD   I0.1
A    SM0.4   // 30sn ON/OFF
=    Q0.1

📋 Bölüm 7 — Özet Kartı

SM0.0: Daima 1 — MOV/blok EN için
SM0.1: İlk tarama — CTD LD, init için
SM0.4: 1 dk flaşör (30s/30s)
SM0.5: 1 sn flaşör (0.5s/0.5s)
SM1.0: Sonuç sıfır
SM1.1: Taşma/geçersiz veri
SM1.2: Negatif sonuç
SM1.3: Sıfıra bölme (kritik!)

I0.0 ve I0.1 kontaklarını etkinleştirerek enerji akışını gözlemleyin.

📋 STL ↔ LAD Karşılaştırmalı — Mühürleme Devresi
🪜 Ladder Diyagramı
📋 STL Karşılığı
STL — Mühürleme 
// I0.0: Start (NO)
// I0.1: Stop (NC → AN kullan)
// Q0.0: Motor

LD    I0.0   // Start NO
O     Q0.0   // Mühürleme (self-hold)
// ↑ Paralel = OR
AN    I0.1   // Stop NC → AN ile
=     Q0.0   // Çıkış

// Set/Reset alternatifi
// (daha okunabilir):
LD    I0.0
S     Q0.0, 1
LD    I0.1
R     Q0.0, 1
⚠ Hata Rehberi

Sık Yapılan Hatalar

S7-200 programlamada en yaygın hatalar, nedenleri ve doğru alternatifler.

Hata Hatalı Kullanım Sorun Doğru Kullanım Bölüm
NOT güç hattında L+ — [NOT] — (Q0.0) Derleme hatası NOT doğrudan L+'ye bağlanamaz Önüne bir kontak koyun: LD I0.0 / NOT / = Q0.0 B.1
Sayıcıya direkt sinyal LD I0.0 / CTU C0, 10 Binlerce hatalı sayım — sensör kapalıyken her taramada artış LD I0.0 / EU (P kenar) / LD ... / CTU C0, 10 B.4
TONR sıfırlanmıyor TONR kullanıp R komutu yok 32767'de kilitlenir — daha fazla sayamaz Devreye mutlaka bağımsız R Tx, 1 kolu ekleyin B.3
MOV EN güç hattında L+ — EN(MOV_W) Derleme hatası — EN doğrudan bağlanamaz LD SM0.0 / MOV_W AIW0, VW100 B.6
CTD LD yükleme yok CTD kullanıp LD sinyali gönderilmemiş Sayıcı sayamaz — başlangıç değeri yok SM0.1 ile LD girişine sinyal gönderin B.4
LPS kapatılmadı LPS kullanıp LPP yok Yığın taşması — alt bitler kaybolur Her LPS için mutlaka bir LPP ile kapatın B.5
Boyut uyumsuzluğu MOV_B AIW0, VW0 Komut çalışmaz — AIW Word, B=Byte uyumsuz MOV_W AIW0, VW0 B.2
Analog tek sayı adres AIW1, AIW3, AIW5… Adres çakışması — analog 2 byte yer alır AIW0, AIW2, AIW4… (çift sayılar) B.2
P/N güç hattında L+ — [P] — (Q0.0) Derleme hatası — NOT gibi aynı kural Önüne bir LD komutu koyun B.1
Sıfıra bölme koruması yok DIV komutu, bölen kontrolsüz SM1.3 set olur — sistem beklenmedik davranış DIV öncesi böleni kontrol edin veya SM1.3 okuyun B.7
